# login-logging Spec Delta

> 新增 capability，初始 spec。包含功能需求和非功能约束。

## 功能需求

### FR-001: 自动记录登录事件
系统在每次用户登录成功后自动记录以下信息：
- 登录时间戳（精确到秒）
- 用户ID
- 来源 IP 地址
- User-Agent 字符串
- 登录是否成功

日志记录必须在登录流程之外异步执行，不得增加登录响应延迟。

### FR-002: 管理端查询登录日志
提供 `GET /api/admin/login-logs` 端点，支持：
- 按时间范围过滤（start_date, end_date）
- 按用户ID过滤
- 分页（offset/limit，limit ≤ 100）
- 返回按时间倒序排列的结果

### FR-003: 特定用户登录历史查询
提供 `GET /api/admin/login-logs/:userId` 端点，返回指定用户的登录历史，支持分页和按时间过滤。

### FR-004: 数据保留策略
登录日志默认保留 90 天，超期数据自动清理（按月分区，删除过期分区）。

## 非功能约束

### NFR-001: 安全与权限
- 登录日志查询接口需要管理员角色权限
- 日志记录不可被修改或删除（append-only, immutable）
- 管理员查询操作本身记录到独立的审计日志

### NFR-002: 性能
- 日志写入异步化，不阻塞登录主流程（写入响应 ≤ 50ms）
- 100万条日志数据下，分页查询响应 ≤ 500ms
- login_time 字段建立 B-tree 索引，user_id 建立 Hash 索引

### NFR-003: 数据保护
- 日志中不包含密码、Token 明文等认证凭证
- IP 地址完整存储但调试日志输出仅保留前8字符哈希
- 数据库连接使用 TLS 加密传输
- 支持用户请求导出/删除自身登录日志数据

### NFR-004: 输入校验
- 所有查询参数进行类型/格式/范围校验
- 使用参数化查询防止 SQL 注入
- IP 字段长度 ≤ 45 字符, User-Agent ≤ 512 字符

### NFR-005: 速率限制
- GET /api/admin/login-logs 每用户每分钟 ≤ 30 次
- GET /api/admin/login-logs/:userId 每用户每分钟 ≤ 60 次
