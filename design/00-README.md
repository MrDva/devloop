## 快速导航

### 5 阶段闭环

```
┌──────────────────────────────────────────────────────────┐
│  ① 逆向分析    ② 需求摄入    ③ 代码生成    ④ 验证交付    │
│  Code→KB→Req   Ext+KB→Req    Req→Code      Verify→Done  │
│      ↑                                        ↓         │
│      └────────── ⑤ 闭环回写 ←────────────────┘          │
└──────────────────────────────────────────────────────────┘
```

### 核心设计原则

1. **Skill 可替换** — 每个阶段通过模板化 Skill 编排，Skill 可插拔
2. **门禁驱动** — 每个产物必经质量门禁校验，不通过不进入下一步
3. **状态持久** — 每步 checkpoint，中断可恢复，不丢上下文
4. **Git 原生** — 一切产物纳入 Git 管理，天然版本控制与保鲜
5. **知识库为核心** — KB 是连接逆向分析与正向生成的桥梁，带防御机制的信任体系
6. **默认高保障** — 确认策略默认 Level 3，用户可主动降级
7. **输出正确性** — 验证聚焦测试通过性，不要求环境一致性

### 关键决策点

- **确认策略**: 默认高保障（Level 3），按需求类型可降低（详见 ADR-4）
- **Git 策略**: 推荐需求与代码同仓库（`/requirements` 目录），简化原子操作
- **知识库格式**: Markdown + frontmatter，带 confidence/drift_score 防御标记
- **状态管理**: `.comet/loop-state.yaml` + Git commit 双保险
- **范围**: 单开发者串行，设计预留并发扩展
- **验证**: 测试通过 = 正确，不要求环境一致
- **保鲜**: Git diff 驱动的增量保鲜 + 每周/每月定时巡检 + 手动触发
