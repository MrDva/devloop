## 快速导航

### 5 阶段闭环

```
┌──────────────────────────────────────────────────────────┐
│  ① 逆向分析    ② 需求摄入    ③ 代码生成    ④ 验证交付    │
│  Code→KB→Req   Ext+KB→Req    Req→Code      Verify→Done  │
│      ↑                                        ↓         │
│      └────────── ⑤ 闭环回写 ←────────────────┘          │
└──────────────────────────────────────────────────────────┘
```

### 核心设计原则

1. **Skill 可替换** — 每个阶段通过模板化 Skill 编排，Skill 可插拔
2. **门禁驱动** — 每个产物必经质量门禁校验，不通过不进入下一步
3. **状态持久** — 每步 checkpoint，中断可恢复，不丢上下文
4. **Git 原生** — 一切产物纳入 Git 管理，天然版本控制与协作
5. **知识库为核心** — KB 是连接逆向分析与正向生成的桥梁

### 关键决策点

- **Git 策略**: 推荐需求与代码同仓库（`/requirements` 目录），简化原子操作
- **知识库格式**: Markdown + frontmatter，人机可读，Git 友好
- **状态管理**: `.comet/loop-state.yaml` + Git commit 双保险
- **人机协作**: 关键决策点暂停等待人工确认，常规流程自动推进
