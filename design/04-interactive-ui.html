需求状态流转
draft→ proposed→ designed→ planned→ building→ built→ verifying→ completed
状态触发条件对应阶段Git 位置
draft外部需求输入到达阶段二开始in-progress/
proposedG2.5 通过阶段二结束in-progress/
designedG3.2 通过 + 人工确认阶段三 Designin-progress/
plannedG3.3 通过 + 人工确认阶段三 Planin-progress/
buildingBuild 模式启动阶段三 Buildin-progress/
builtG3.4 通过阶段三结束in-progress/
verifying阶段四开始阶段四in-progress/
completedG4.4 通过阶段四结束completed/
blocked门禁持续失败 / 人工操作任意阶段in-progress/
异常路径
  • draft → rejected: G2.1 失败（不可恢复的格式错误）
  • 任何一个状态 → blocked: 门禁持续失败 / 人工阻塞
  • verifying → fixing → verifying: 验证失败修复循环 (最多3次)
  • blocked → proposed: 人工解除阻塞后重新流入
  • blocked → [废弃]: 需求取消
